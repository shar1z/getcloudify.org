Как это работает

  • 1. Конфигурация плана оркестрации
  • 2. Установка одним щелчком
  • 3. Мониторинг
  • 4. Автовосстановление
  • 5. Обновление
  • 6. Автомасштабирование

Перевод существующих приложений в облако без изменения кода легко выполняется с помощью Cloudify. Секретом этого является план оркестрации. План оркестрации Cloudify - это простая конфигурация, которая точно описывает развертывание приложения и план автоматизации управления в любом облаке.

Планы оркестрации Cloudify конфигурируют зависимости между уровнями приложения, события жизненного цикла (установка, запуск, конфигурация, остановка, удаление и т.д.) по уровням и облачные ресурсы, используемые каждым уровнем. Кроме этого, инструкция определяет возможность контроля, обновления, поддержки и масштабирования уровней приложения по мере необходимости.

Оркестратор использует предопределенный план оркестрации для непрерывной установки вашего приложения в облаке: от предоставления виртуальных машин до развертывания кода приложения и динамического подключения компонентов вашего приложения, беря во внимание зависимости между ними.

Cloudify применяет определённый облачный API посредством уровня облачного драйвера. Драйвер облака создает виртуальные машины и устанавливает агенты Cloudify на каждой из них. После этого агенты Cloudify разворачивают и запускают ваши приложения в соответствии с планом оркестрации.

Cloudify не прекращает свою работу после завершения развертывания. Cloudify контролирует ваше приложение на основе пользовательских метрик, которые вы определили. Получите самые современные индикаторы производительности на удобной панели управления.

Cloudify позволяет вам подключиться к технологии мониторинга вашего выбора, предоставляет средства для установки агентов мониторинга, как часть установки приложения, и передает ваши метрики в механизм политик Cloudify и базу данных метрик. Cloudify визуализирует ваши метрики мониторинга и статистические данные с помощью веб-отчетов и предоставляет вам API для запроса метрик мониторинга из любых ваших систем.

Новая улучшенная технология автоматического восстановления Cloudify позволяет вам получить максимальную пользу от облака с помощью поддержки работоспособности приложения без ручного вмешательства.

Cloudify автоматически восстанавливает неисправное ПО промежуточного слоя или потерянные виртуальные машины при автоматической перенастройке приложения. Продолжайте следить за нашими следующими версиями и всеми будущими улучшениями.

После фазы развертывания Cloudify поддерживает обновления и точную настройку приложений посредством пользовательских рабочих потоков. Рабочие потоки могут быть запущенны вручную или программным способом.

Графический пользовательский интерфейс (GUI), интерфейс командной строки (CLI) и REST API Cloudify позволяют вам активизировать собственный рабочий поток, описанный в плане оркестрации. Например, ваш рабочий процесс может определить, как получить новую версию вашего кода и развернуть его в ваше ПО промежуточного слоя.

Cloudify будет автоматически масштабировать ваше приложение, используя пользовательские политики с правилами масштабирования, запуск которых основан на определенных вами метриках.

Когда оркестратор Cloudify обнаруживает нарушение SLA с помощью пользовательских правил масштабирования, он активизирует горизонтально-масштабируемый рабочий поток, который может создать дополнительные виртуальные машины, добавить их в балансировщик нагрузки или выполнить любой другой набор необходимых шагов (с помощью пользовательского рабочего потока).
