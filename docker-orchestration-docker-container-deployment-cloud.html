Представляем оркестрацию Docker с плагином Cloudify 3.0

Встроенное управление и поддержка Docker

Cloudify имеет встроенный Docker плагин, который позволяет пользователям использовать контейнеры Docker как часть любой топологии. Это предоставляет полнофункциональный метод оркестрации и автоматизации для крупномасштабных систем на основе Docker. Кроме этого, готовые образы Docker, для развертывания вашей среды управления, обеспечивают встроенную поддержку различных дистрибутивов Linux, более легкие и быстрые обновления ваших сред управления, а также более быструю начальную загрузку.

Интеллектуальная оркестрация Docker - объединение всех частей в одно целое

  • Контейнеры Docker были созданы, чтобы выполнять быстрое и надежное развертывание компонентов или уровней приложения с помощью создания контейнера, который содержит готовые к развертыванию автономные части приложений с ПО промежуточного слоя и бизнес-логикой приложения, необходимые для успешного запуска. Docker преднамеренно изолирует автономные части приложения, обычно один уровень или даже узел уровня.
  • Однако приложение часто имеет многоуровневую архитектуру. Это означает, что есть уровни с зависимостями между ними, где характер зависимостей может быть разным: от сетевых соединений и удаленных вызовов API, до обмена сообщениями между уровнями приложения. Таким образом, приложение является набором различных контейнеров со специальными конфигурациями. Поэтому необходим способ объединения всех частей вашего приложения.
  • Именно здесь вступает в дело Cloudify - с помощью оркестрации Cloudify Docker мы позаботимся о создании контейнера в нужное время, полагаясь на порядок зависимостей, а также о создании всех необходимых конфигураций, чтобы позволить контейнерам взаимодействовать и передавать друг другу необходимые свойства времени выполнения.
  • Основываясь на TOSCA и с помощью плана оркестрации Cloudify в формате YAML, вы можете описать даже самые комплексные топологии, включая: инфраструктуру, уровень ПО промежуточного слоя и различные уровни приложения. Cloudify просто берет план оркестрации TOSCA и осуществляет его с помощью рабочих потоков, которые проходят по графу компонентов или плану компонентов и передают команды агентам. Затем агенты создают эти компоненты и объединяют их в приложение.
